Modelo de Caixa (Box Model)

Se você acompanhou a seção de HTML, viu que usamos tags para estruturar o conteúdo. O que talvez você não tenha percebido é que, para o navegador, cada uma daquelas tags é, na verdade, uma caixa.

Desde um simples <span> até um grande <main>, tudo o que você coloca na tela é tratado como um retângulo. O Box Model é a regra do CSS que define como o tamanho dessas caixas é calculado e como elas interagem umas com as outras.

Mas, antes de entendermos como essas caixas se comportam, precisamos aprender a "conversar" com o código de forma organizada.

Notas no Código: O poder dos Comentários

No CSS, usamos /* */ para escrever anotações que o navegador ignora.

Por que isso é importante? Porque o CSS pode ficar gigante. Comentários servem para separar seções (ex: /* Estilo do Menu */), explicar por que você usou um valor específico ou até "esconder" uma regra temporariamente para testes. Um código bem comentado é o sinal de um desenvolvedor profissional.

As 4 Camadas da Caixa

Imagine que cada elemento é uma caixa de presente protegida por várias camadas:

  1. Content (Conteúdo): É o objeto dentro da caixa (seu texto, link ou imagem). É a área onde o conteúdo realmente aparece.
  2. Padding (Preenchimento): É como se fosse o plástico bolha. É o espaço interno que protege o conteúdo da borda. Se você quer que o texto não fique "colado" na linha da caixa, você aumenta o padding.
  3. Border (Borda): É a própria caixa de papelão. Ela envolve o conteúdo e o preenchimento. Pode ser fina, grossa, colorida ou até invisível.
  4. Margin (Margem): É a distância de segurança entre a sua caixa e a caixa do vizinho. A margem é sempre transparente e serve para empurrar os outros elementos para longe.

A Anatomia no Código

No CSS, controlamos cada parte assim:

    div {
        box-sizing: border-box; /* Largura e altura total*/

        height: 300px;          /* Altura do Conteúdo */
        width: 300px;           /* Largura do Conteúdo */

        margin: 50px;           /* Distância externa */
        padding: 20px;          /* Espaço interno */

        border: 5px solid;      /* Espessura da borda */
        }

O Controle Total com Box-sizing

Por padrão, o navegador é um péssimo matemático. Se você define uma largura e depois adiciona bordas, ele soma tudo e sua caixa fica maior do que o planejado.

  • box-sizing: content-box;: É o padrão (e o vilão). A largura que você define vale apenas para o conteúdo. Padding e bordas são somados por fora.
  • box-sizing: border-box;: É o herói. Ele diz ao navegador: "A largura total é esta e ponto final". O padding e a borda são espremidos para dentro, mantendo o tamanho que você definiu.

Dominando o Tamanho (Width e Height)

No CSS, definir o tamanho de uma caixa não é apenas sobre números, é sobre comportamento. Existem dois tipos de unidades: as Absolutas (que não mudam nunca) e as Relativas (que dependem de algum fator externo).

  1. Unidades Absolutas: O Pixel (px)

    O pixel é a unidade mais comum. Ele é fixo. Se você define width: 300px, aquela caixa terá exatamente esse tamanho, seja em um monitor gigante ou em um celular pequeno.

    • Vantagem: Precisão total. Ótimo para bordas ou elementos que não podem mudar de tamanho.
    • Desvantagem: Não é responsivo. Pode fazer o seu site ter aquela barra de rolagem horizontal chata em telas menores.
  2. Unidades Relativas: Porcentagem (%)

    A porcentagem é relativa ao elemento pai. Se você tem uma div dentro do body e define width: 50%, ela ocupará metade da largura disponível do corpo da página.

    • Uso ideal: Criar colunas e layouts que se ajustam quando você redimensiona a janela do navegador.
  3. Unidades de Viewport: vw e vh

    Essas são as unidades mais modernas e poderosas. Elas são relativas ao tamanho da janela do navegador (viewport), ignorando quem é o "pai" do elemento.

    • vw (Viewport Width): 1vw é igual a 1% da largura total da tela. 100vw ocupa a tela inteira de ponta a ponta.
    • vh (Viewport Height): 1vh é igual a 1% da altura total da tela.
    • Exemplo Prático: Quer que sua seção de "Destaque" ocupe a tela inteira do usuário assim que ele abre o site? Use height: 100vh;.
  4. O "Coringa" do Tamanho: max-width

    Em vez de usar apenas width: 500px, os devs profissionais costumam usar max-width: 100%.

    • O que isso faz? Diz ao navegador: "Pode ser até 500px, mas se a tela for menor que isso, diminua para caber 100% no espaço disponível". Isso evita que seu layout quebre em celulares.

Espaçamentos (Margin e Padding)

É aqui que o site ganha vida e para de parecer um amontoado de coisas coladas.

  • Padding (Interno): É a distância entre o conteúdo e a borda. Aumenta a área de clique de um botão, por exemplo.
  • Margin (Externo): É o espaço entre a sua borda e os outros elementos ao redor.
  • O Truque do margin: auto;: Quando você define uma largura fixa para uma caixa (ex: width: 500px;) e aplica margin: auto;, o navegador distribui o espaço disponível igualmente dos dois lados, centralizando o elemento horizontalmente na tela. É o jeito mais clássico de centralizar um site!
