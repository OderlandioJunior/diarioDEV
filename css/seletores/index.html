Seletor CSS

Se o HTML é o corpo do seu site, o CSS é a roupa e a maquiagem. Mas, para vestir a roupa certa na pessoa certa, você precisa apontar para ela. O Seletor é exatamente esse "dedo apontado". É a instrução que diz ao navegador: "Pegue este elemento específico no HTML e aplique estas regras a ele".

A Anatomia de uma Regra CSS

Antes de mergulhar nos tipos, você precisa entender como se escreve uma regra. Ela é composta por:

  • Seletor: O alvo (ex: h1).
  • Declaração: O conjunto de estilo, que fica entre chaves { }.
  • Propriedade: O que você quer mudar (ex: color).
  • Valor: A nova característica (ex: blue).
    • Exemplo:
    h1 { color: blue; } -> Aqui, h1 é o seletor.

Tipos de Seletores (A Hierarquia)

Existem várias formas de selecionar um elemento. Vamos das mais comuns para as mais específicas:

  1. Seletor de Tipo (ou Tag)

    2. Seleciona todos os elementos de uma mesma tag HTML.

    • Exemplo: p { ... }
    • Efeito: Todos os parágrafos do seu site ficarão iguais.
  2. Seletor de Classe (.)
    3. Seleciona elementos que possuem o atributo class. É o mais usado no dia a dia porque é reutilizável.
    • Exemplo: .botao-azul { ... }
    • No HTML: <button class="botao-azul">Clique aqui</button>
  3. Seletor de ID (#)
    4. Seleciona um elemento único que possui o atributo id.
    • Exemplo: #cabecalho-principal { ... }
    • Regra de Ouro: Um ID deve ser único na página. É como o CPF do elemento.
  4. Seletor Universal (*)

    5. Seleciona absolutamente todos os elementos da página. Muito usado para fazer "resets" de margens e preenchimentos.

Especificidade: A Lei do Mais Forte

O que acontece se você disser no CSS que o parágrafo é azul, mas em outra linha disser que a classe daquele mesmo parágrafo é vermelha? Quem ganha?

É aqui que entra a Especificidade. O navegador usa um sistema de pontos para decidir qual regra vence:

  1. Inline Style (1000 pontos): Estilo escrito direto na tag HTML (style="..."). É o "chefe supremo", mas deve ser evitado por dificultar a manutenção.
  2. ID (100 pontos): Muito forte. Ganha de classes e tags.
  3. Classe, Pseudo-classe e Atributos (10 pontos): Força intermediária.
  4. Tag e Pseudo-elementos (1 ponto): A base da pirâmide, a força mais fraca.
Exemplo Prático:
  • p { color: blue; } (1 ponto)
  • .texto { color: red; } (10 pontos)
  • #titulo { color: green; } (100 pontos)
Se um elemento tiver o ID titulo e a classe texto, ele será verde, porque o ID tem mais "peso" (100 vs 10).

Agrupamento e Encadeamento

Para manter seu código limpo, você pode:

  • Agrupar: h1, h2, p { font-family: sans-serif; } (Usa a vírgula para aplicar a mesma regra a vários seletores).
  • Encadear (Seletores de Descendente): article p { color: gray; } (Seleciona apenas os parágrafos que estão dentro de um article).
