Arquitetura de Dados: Construção de Tabelas em HTML5

As tabelas são utilizadas exclusivamente para exibir dados que possuem uma relação lógica entre linhas e colunas (como cronogramas, listas de preços ou especificações técnicas). Antigamente, tabelas eram usadas para criar o layout dos sites, mas hoje isso é considerado uma prática amadora e incorreta — layouts são responsabilidade do CSS.

A Anatomia Estrutural

Para uma tabela ser semântica e acessível, dividimos sua anatomia em três partes principais:

  • <table>: O container que define o início e o fim da tabela.
  • <thead>: O cabeçalho da tabela. É aqui que definimos os títulos das colunas.
  • <tbody>: O corpo da tabela, onde os dados reais (o conteúdo) ficam armazenados.
  • <tfoot>: O rodapé da tabela, geralmente usado para somatórios ou notas de rodapé.

As Tags de Células e Linhas

Dentro dessas divisões, utilizamos as tags que constroem a grade:

  • <tr> (Table Row): Representa uma linha horizontal completa.
  • <th> (Table Header): Uma célula de cabeçalho. Por padrão, o navegador a deixa em negrito e centralizada.
  • <td> (Table Data): Uma célula de dado comum, que compõe o corpo da tabela.

Extensão de células

Quando estamos lidando com dados mais complexos, nem sempre uma informação cabe em apenas um "quadradinho" da tabela. É aí que entram os atributos de expansão (ou mesclagem), que permitem que uma célula ocupe o espaço de várias outras.

No HTML, usamos o termo span (que significa "alcance" ou "extensão") para esticar as células em duas direções:

Mesclagem Horizontal: colspan

O atributo colspan (abreviação de column span) define quantas colunas uma célula deve ocupar para a direita.

  • Quando usar: Imagine que você quer criar um título que cubra toda a largura da tabela ou uma categoria que englobe vários subitens laterais.
  • Regra de Ouro: Se você definir colspan="2", você deve "remover" uma tag <td> ou <th> daquela mesma linha, caso contrário, a tabela vai ficar com uma célula sobrando para fora da grade.

Mesclagem Vertical: rowspan

O atributo rowspan define quantas linhas uma célula deve ocupar para baixo.

  • Quando usar: É perfeito para quando um mesmo dado se repete em várias linhas consecutivas. Em vez de escrever "HTML5" três vezes em três linhas diferentes, você cria uma célula única que "estica" para baixo.
  • Regra de Ouro: Esta é a que mais confunde! Se você usar rowspan="3" na primeira linha, você não deve criar a primeira célula das próximas duas linhas, pois o espaço delas já foi "roubado" pela célula que veio de cima.

Semântica e Acessibilidade: O atributo caption

Toda tabela profissional deve ter um "título" visível que descreva seu conteúdo. Para isso, utilizamos a tag <caption>, que deve ser a primeira tag logo após a abertura da <table>. Isso ajuda usuários de leitores de tela a entenderem rapidamente do que se trata aquele conjunto de dados.
